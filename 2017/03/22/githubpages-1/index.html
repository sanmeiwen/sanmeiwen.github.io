用 github pages 和 hexo创建个人博客（一）——简单介绍

2017-03-22
  • IT

前言

看惯了别人都有博客，不管是不是大牛，但是都可以拿来装 B（静静的装）。想到这里就对了，Follow me，本系列文章将带你走上从此装 B 一去不返的“致富”之路。

像我这样喜欢记录或者写博客的人大致都会经历这么三个阶段。

第一阶段：初恋的感觉

刚接触Blog，觉得很新鲜，就像想要恋爱一样，然后就试着找一个对象谈恋爱，找了好久，仔细对比了一下CSDN、博客园、IT eyes、blog java和开源中国，最终还是选择了一个大众情人——CSDN。

  • CSDN : 从我的感觉来说，整体可以，也是我用的最久的（初恋嘛，你懂得……），但是也有毛病，因为之前格式不好看，上传图片麻烦，自定义程度太低了。后来推出了 markdown 格式，但是主题还是太少了，最终还是分手了……
  • ITeye : 主要是 java 博客，自从用的java不多的时候都很少上这个网站了，更别说是写博客了，写了几篇就没写了（主要是因为浏览的人太少了——写的太烂）。
  • 其他的用的就更少了，这里就不提了……

初恋总的来时还是很好用的，毕竟憋了好多年，怎么可能不好用嘛，想想假如你自己三天没吃饭了，给你狗粮，你都会觉得倍儿香（此处省略 47 个字）……

第二阶段：自力更生

跟免费的 Blog 空间谈了也是蛮久的恋爱，但是越来越发现便宜的东西不好用，限制还多的要死，我们都是向往自由的，所以分手了，自力更生，可累坏了我这一双手了，心疼啊。

后来就自己买域名，买空间，而且 WordPress 那么有名，就用她搭建了自己的博客系统，而且也用了好长一段时间，但是学生期间，没有太多的钱，总不能去卖肾或者裸贷吧（裸贷人家估计也不给贷吧……），后来就没办法，没有再续了。

虽然这一阶段也结束了，但是感觉比第一阶段还要好一些，虽然没有初恋用了，都要靠自己的双手，但是依然可以解决问题，重要的是更快更舒服……

第三阶段：安安静静的做一个只写东西的“美男子”

自力更生固然不错，但是，一旦自己有事，而这个时候服务器还出问题了，这个就像你已经忙的一个月没有解决生理问题了，晚上突然特别想的时候，没有女票，手还受伤了，不能动，嘴又够不上，此时……

所以，现在想着如果能有人帮我解决了生理问题，但是一天还不管我，我想干什么就干什么，多么美好啊（千万不能让我女票看到，不然晚上又得跪键盘，睡沙发了）。慢慢觉得独立博客的管理太麻烦，最好在保留控制权的前提下，让别人来管，自己只负责写文章。

上面提到了两个关键点

  • 忙：忙导致没时间管一些除了主要事情之外的一些东西。
  • 想的美：想要有女朋友，而且还要无拘无束的生活，你是不是想得美。

有没有解决这个问题的办法呢？当然有，伟大的 github 推出了 pages 服务（码云也有），她可以托管静态网站，但是静态的对只写东西的人来说不就够了吗？她还允许我们自定义主题，她就解决了我上面的两个关键问题，那我肯定用她啊。

既然女朋友（代码托管的地方——github pages）已经选好了，那么接下来该带去整容了，不然咋看啊，虽说晚上灯一关都一样，但是一想起来就软了……

jelly 是 github pages 最好的搭配，但是我没有找到好看的脸型（主题，虽然我在菊花厂，但我还是比较喜欢 Google 的 Material Design），最后在 hexo 找到了这款主题，于是就用了这个脸型，可见女生漂亮是有多重要啊。

介绍

下面就开始介绍我选的女票和整的容了，如果你也跟我有同样的想法，我借你用几天可好……

Github

github

GitHub 是一个面向开源及私有软件项目的托管平台，因为只支持 Git 作为唯一的版本库格式进行托管，故名 GitHub。作为开源代码库以及版本控制系统，Github 拥有超过900万开发者用户。随着越来越多的应用程序转移到了云上，Github已经成为了管理软件开发以及发现已有代码的首选方法。

github pages

为使项目更方便的被人理解，介绍页面少不了，甚至会需要完整的文档站，Github 替你想到了这一点，他提供了 Github Pages 的服务，不仅可以方便的为项目建立介绍站点，也可以用来建立个人博客。

Github Pages 有以下几个优点：

  • 轻量级的博客系统，没有麻烦的配置
  • 使用标记语言，比如 Markdown
  • 无需自己搭建服务器
  • 根据 Github 的限制，对应的每个站有 300MB 空间
  • 可以绑定自己的域名

hexo

按照官网的说法，Hexo是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

Hexo + Github

通过上面的介绍，大家对Hexo和Github已经有了大概的了解。
没错，我们的方式就是，利用Markdown进行博客的编写，通过Hexo这个框架解析生成靓丽的静态页面，然后部署到Github上供大家浏览。

这种方法具备好处是:

  • 免费，无限流量。
  • 享受git的版本管理功能，不用担心文章遗失。
  • 专注于我们自己的博客内容，其他诸如服务器管理的事情交给Github即可

当然啦，万事有利有弊。这种方式也有它的弊端:

  • 有一定技术门槛，你必须要懂一点git和网页开发。
  • 生成的是静态网页，添加动态功能必须使用外部服务，比如评论功能我们使用多说。
  • 不适合大型网站，因为没有用到数据库，每运行一次都必须遍历全部的文本文件，网站越大，生成时间越长。

但是，这已经基本满足我们第三阶段的需求啦。
